# v0.8.1

## Fixed

Two `Cdn` settings-update bugs that failed with `400 bad_request` on resources
declaring a `performance` block and/or two custom delivery domains (v0.8.0):

- **`config.delivery.packaging.mp4`** was sent as a boolean; the upstream
  requires an object or null. The provider now sends the video-packaging field
  only when its state changes, never as a bool.
- **`config.domains.aliases`** included the technical domain, which the upstream
  counts inside its 2-alias limit (2 custom + technical = 3 → rejected). The
  write now carries only the declared custom domains.
