# v0.3.2

## Added

- **Nodepool worker flavor.** `KubernetesClusterNodepool.spec.forProvider.resources.flavor`
  selects the worker configurator family: `standard` (default) → general
  (`k8s_configurator_general`), `dedicated-cpu` → dedicated-CPU
  (`k8s_configurator_dedicated_cpu`). Resolver now pins the family before the
  tightest-fit sort, so it no longer silently picks dedicated-CPU and rejects small
  ratios (e.g. 2 cpu / 2 GB) with `invalid_configuration_ram`. Existing pools keep their
  locked configurator; omitting `flavor` resolves to general.
