# v0.4.1

## Added

- **`S3User` connection Secret: `buckets` key.** A comma-separated, sorted list of **every**
  bucket the user can reach. The same `access_key`/`secret_key` + `endpoint` authorize all of
  them; previously only the first bucket was discoverable via the singular `bucket` key.
  - `bucket` (singular) is unchanged — it still names the primary (first, sorted) bucket — so
    existing single-bucket consumers are unaffected.
  - Multi-bucket consumers should read `buckets` (e.g. `IFS=, read -ra BUCKETS <<<"$buckets"`).
