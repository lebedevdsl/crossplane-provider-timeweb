# v0.8.2

## Fixed

- **`Cdn` custom SSL certificates no longer use a retry budget.** Custom upload
  is deterministic, so the Let's-Encrypt-style budget was wrong there — it left
  a successfully bound custom certificate showing a stale
  `ssl.state: failed` / `issueAttempts: 1`. A custom certificate is now
  uploaded once per distinct identity; a bound cert reads `ssl.state: bound`,
  and a rejected one stays `failed` without re-uploading until the referenced
  Secret changes. `issueAttempts` / `lastIssueAttemptAt` are Let's-Encrypt-only.
