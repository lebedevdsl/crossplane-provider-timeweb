# v0.6.0

## Added

- **Nodepool taints** — `KubernetesClusterNodepool.spec.forProvider.taints[]`
  (`key` / optional `value` / `effect: NoSchedule|PreferNoSchedule|NoExecute`,
  ≤12 entries). Nodes join the cluster already tainted — no post-join window.
  - Admission validation: effect enum, Kubernetes label-syntax key/value
    patterns, duplicate key+effect rejected (same key with different effects
    is allowed).
  - Every node added later (scale-up, autoscaler, autohealing) carries the
    group's current sets.
- **`status.atProvider.labels` / `.taints`** — mirror of what the upstream
  group actually reports, in spec shape, refreshed on every observation.

## Changed

- **Nodepool `labels` and `taints` are day-2 mutable** (labels were
  create-only before). Edits converge in place via a group PATCH — running
  nodes are re-configured by the platform without recreation (they pass
  through an `updating` state), and metadata converges on autoscaled pools
  too. The declaration is the single writer: panel/API edits to the group's
  labels or taints are reverted on the next reconcile. Removing the blocks
  clears the sets upstream.

## Known platform quirks

- A taint with an **empty value** persists on the upstream group but is not
  applied to the node objects — give every taint a value. (The PATCH wire
  format requires the `value` key; the provider always sends one.)

## Docs
