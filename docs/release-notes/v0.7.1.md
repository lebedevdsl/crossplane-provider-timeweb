# v0.7.1

## Added

**Per-parameter query-string cache keys** on `Cdn` — key the CDN cache on
specific query parameters instead of all-or-nothing (wire-verified
`query_args: {mode, list}`):

```yaml
spec:
  forProvider:
    origin:
      bucketRef: { name: site-assets }
      https: true
    cache:
      edgeTTLSeconds: 86400
      queryStringCacheKeyMode: whitelist    # all | whitelist | blacklist
      queryStringCacheKeyParams: ["utm_source", "ref", "v"]
```

- `whitelist` = only the listed parameters split the cache; `blacklist` = all
  except them.
- Mutually exclusive with the existing `queryStringInCacheKey` bool
  (admission-enforced); the bool remains the shorthand for "all".
- `Cdn` added to the README kinds table; bundle `23-cdn` documented in the
  e2e README.

Field reference:
[docs/cdn.md](https://github.com/lebedevdsl/crossplane-provider-timeweb/blob/v0.7.1/docs/cdn.md).
