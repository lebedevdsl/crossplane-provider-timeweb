# v0.8.0

## Added

**`Cdn` — the full remaining panel surface**: custom delivery domains, SSL
certificate management, signed-URL secure token, outbound traffic limit, and
AWS auth for external S3 origins.

```yaml
spec:
  forProvider:
    origin:
      bucketRef: { name: site-assets }
      https: true
    domains: [cdn.example.com]        # ≤2; CNAME → technicalDomain (your DNS)
    ssl:
      mode: custom                    # none | letsEncrypt | custom
      certificateSecretRef: { name: example-tls }   # kubernetes.io/tls
    security:
      forceHTTPS: true
      secureToken:
        secretRef: { name: cdn-signing }
        restrictByIP: true
    trafficLimitGBPerMonth: 3000      # GiB; exceeding suspends (≤2 h lag)
```

```sh
# reset the Let's Encrypt retry budget after fixing DNS:
kubectl annotate cdn/site-assets cdn.timeweb.crossplane.io/retry-ssl=now
```

- **`ssl.mode: letsEncrypt`** issues and binds automatically once the custom
  domain's CNAME is live (verified end-to-end). Issuance can fail transiently
  with no error reason while DNS propagates to Let's Encrypt; retries are
  budgeted (4 × ≥15 min → `ssl.state: exhausted`, resettable via a spec edit
  or the `retry-ssl` annotation). Custom certificates must chain to a
  system-trusted root (self-signed is rejected upstream).
- Custom-cert rotation is declarative: update the TLS Secret → upload, rebind,
  old managed certificate removed. The provider never deletes certificates it
  didn't create.
- Removing `secureToken` disables signed URLs; the key never leaves the Secret.
- The delivery endpoint is still published to the connection secret
  (`technical_domain`, `url`) for app/DNS wiring.

Field reference and the signing algorithm:
[docs/cdn.md](https://github.com/lebedevdsl/crossplane-provider-timeweb/blob/v0.8.0/docs/cdn.md).
