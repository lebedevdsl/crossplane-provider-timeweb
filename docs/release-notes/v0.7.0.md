# v0.7.0

## Added

- **`Cdn` kind** (`cdn.m.timeweb.crossplane.io/v1alpha1`, new group) — a Timeweb
  Cloud CDN resource: one origin behind an auto-assigned technical delivery
  domain (`status.atProvider.technicalDomain`).
  - Origin: exactly one of `bucketRef` (an `S3Bucket` in the same namespace,
    attached by upstream storage id; creation waits on bucket readiness),
    `domain`, or `ip`, plus an `https` scheme toggle and optional `port`.
    Domain origins must resolve in public DNS (the upstream rejects
    non-resolvable hosts).
  - Declared settings blocks are owned single-writer with drift reversion:
    `cache` (edge TTL, browser TTL, always-online, query-string cache key),
    `security.forceHTTPS`, `performance` (HTTP/3, gzip, large-file slicing
    1–1024 MB, content optimization `off|video|images`, robots.txt
    `deny|proxy|custom`), `cors`, `requestHeaders`. Omitted blocks stay
    panel-managed and are mirrored read-only under
    `status.atProvider.observedSettings`.
  - Cache purge via a self-clearing annotation:
    `cdn.timeweb.crossplane.io/purge=all` (full) or comma-separated `/`-rooted
    paths (selective). One purge per annotation set; completion recorded as
    `status.atProvider.lastPurgedAt`, audit trail in Events.
  - Billing preset resolved automatically at create and locked in status; no
    operator-facing tier field.
  - Updates converge against the configuration readback (one PATCH per
    reconcile); the unreliable upstream `status` field is mirrored but never
    gates Ready, updates, or purges — only a suspended state does.

## Known platform quirks

- The upstream `status` sticks at `processing` for hours (terminal value is
  `created`) on resources that serve, apply changes, and purge normally; the panel's
  "Применяются настройки" badge and purge spinner track the same stuck field.
- A non-resolvable origin host fails the create with a bare
  `500 internal_server_error` (no message) — surfaced as a retrying
  `Synced=False APIError`; check the origin spelling.
- `clear-cache` can return 500 in the first moments after creation and 429
  (`rate_limit_exceeded`) on repeated purges — the controller retries with the
  purge annotation retained until it succeeds.
- The CDN API is undocumented (`/api/v1/cdn/http-resources`); endpoint
  inventory and verified payloads are recorded in
  `specs/016-cdn-resource/contracts/timeweb-cdn-endpoints.md`.
