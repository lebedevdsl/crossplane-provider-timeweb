# v0.7.0

## Added

**`Cdn`** (`cdn.m.timeweb.crossplane.io/v1alpha1`) — a Timeweb Cloud CDN
resource: one origin served through an auto-assigned delivery domain, declared
settings kept in sync, cache purge by annotation.

```yaml
apiVersion: cdn.m.timeweb.crossplane.io/v1alpha1
kind: Cdn
metadata:
  name: site-assets
  namespace: web
spec:
  forProvider:
    origin:
      bucketRef: { name: site-assets }   # or domain: / ip: — exactly one
      https: true
    cache: { edgeTTLSeconds: 86400, queryStringInCacheKey: false }
    security: { forceHTTPS: true }
    performance: { gzip: true, http3: true }
  providerConfigRef:
    kind: ProviderConfig
    name: default
```

Cache purge — one-shot, the annotation clears itself when the purge lands:

```sh
kubectl annotate cdn/site-assets cdn.timeweb.crossplane.io/purge=all
kubectl annotate cdn/site-assets cdn.timeweb.crossplane.io/purge=/,/img,/index.html
```

- Delivery domain: `status.atProvider.technicalDomain`.
- Settings blocks you omit stay panel-managed (mirrored read-only in status).
- Private-bucket origins need no credentials in the manifest.

Field reference, conditions, and platform quirks:
[docs/cdn.md](https://github.com/lebedevdsl/crossplane-provider-timeweb/blob/v0.7.0/docs/cdn.md).
