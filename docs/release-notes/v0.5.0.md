# v0.5.0

## Added

- **`Firewall` kind** (`network.m.timeweb.crossplane.io/v1alpha1`) — a Timeweb Cloud
  firewall **rule group** managed declaratively.
  - **Policy:** `DROP` (default-deny allow-list, the default) or `ACCEPT`; create-only.
  - **Rules:** inline `rules[]` (`direction` ingress|egress / `protocol` tcp|udp|icmp /
    `port` single or range / `cidr`), set semantics — unordered, converged by add/remove
    diff, ≤128 rules.
  - **Attachments:** inline `attachedServices[]` by opaque `{serviceID, serviceType}`
    (`server|dbaas|balancer|app`, default `balancer`) — v1 targets load balancers.
    A service belongs to at most one group (`ServiceConflict` otherwise); this resource
    never silently steals a service bound elsewhere.
  - **Single writer:** dashboard edits are reverted to the declared state on the next
    reconcile.
  - **Print columns:** `POLICY`, `RULES`, `ATTACHED`.

## Changed

- **Print columns rationalized** (the live `kubectl get` output drifted from the
  documented house rules):
  - `Network`: `ID` and `STATE` are wide-only (`-o wide`), `AGE` last; the stray
    `TYPE` column is gone.
  - `KubernetesCluster` / `KubernetesClusterNodepool`: `PRESET` (blank for
    resources-sized objects) replaced by `SIZING` — a new controller-written
    `status.atProvider.sizing` summary: `preset:<slug>` or `<cpu>cpu/<ram>gb/<disk>gb`.
  - `KubernetesClusterNodepool` / `KubernetesClusterAddon`: `CLUSTER` now shows the
    parent CR **name** (`clusterRef.name`); the numeric upstream id moved to the
    wide-only `CLUSTER-ID`.
  - `ProviderConfig` / `ClusterProviderConfig`: `SECRET-NAME` visible by default on
    both; `AGE` last.
