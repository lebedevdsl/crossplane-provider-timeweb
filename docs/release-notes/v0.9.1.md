# v0.9.1

Bugfix. Non-breaking; no CRD/API change, no connection-secret changes.

## Fixed

- **False not-found → resource recreation (postmortem #124).** A single flaky/edge HTTP 404
  — the Timeweb API sits behind Qrator DDoS protection and intermittently returns edge 404s —
  was classified as "resource deleted" by HTTP status alone, in the one shared path every
  managed kind's `Observe` routes through. That misclassification recreated a live production
  VPC as an empty duplicate and orphaned the original. A 404 is now recognized as a deletion
  **only** when it carries the canonical Timeweb error envelope (`error_code` present); an
  envelope-less 404 (HTML, empty, or non-envelope JSON) is treated as a transient upstream flap
  and requeued — never recreated. Applies uniformly across every kind via the shared classifier;
  no per-kind changes. Genuine deletions (which carry the envelope) are still recognized exactly
  as before.

See [docs/error-classification.md](https://github.com/lebedevdsl/crossplane-provider-timeweb/blob/v0.9.1/docs/error-classification.md)
for the canonical-not-found rule and the per-API signal table.
