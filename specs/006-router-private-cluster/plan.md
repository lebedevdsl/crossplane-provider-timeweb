## Summary

Add a namespaced `Router` managed resource (`network.m.timeweb.crossplane.io`) for
Timeweb's NAT/DHCP router appliance — inline per-attachment network model
(`nat`/`dhcp`, minItems=1), public addresses by **referencing** the existing
`FloatingIP` kind, tier/preset sizing (tier carries HA + region), and the
**private-cluster arrangement** (US3): a cluster network behind a NAT'd router so
worker nodes need no public IPs. Router lifecycle (US1) and live attachment/NAT
convergence (US2) are e2e-validated in concept; the **NAT activation toggle (T020)
is now implementable** from the official spec and is the load-bearing remaining
piece — without it NAT never activates and US3 cannot function. Public-by-default
for nodes is unchanged (FR-008/SC-006).

### Open clarifications (NEEDS CLARIFICATION → resolved in Phase 0 research)

- **US3 mechanism — DECIDED (2026-06-17, revised): KEEP the explicit `publicIP`
  flag** (Azure-AKS `enableNodePublicIP` precedent; explicit intent beats
  implicit-by-placement). `public_ip_enabled` is re-added to `NodeGroupIn` as a
  spec hand-patch (live-but-unswaggered); the MR keeps `publicIP *bool`. Router-NAT
  + default route remains the orthogonal **egress** mechanism (even public-IP-less
  nodes need NAT to reach the internet); the flag controls public **addressing**.
  Contingent on a live re-probe confirming the create honors the field — validated
  in bundle 19 (`publicIP:false` ⇒ nodes carry no public address). Only if the
  re-probe shows it's ignored do we fall back to placement + document.
- **NAT param identity**: official path is `…/networks/{network_name}` (the
  hand-patch used `{network_id}`). NEEDS confirmation whether NAT/detach key on the
  network **name** or **id** (the controller currently passes the `network-xxxx` id).
- **Router→service binding**: `RouterIn.parent_service{id,type}` exists at create —
  re-evaluate US3 binding via explicit create-time `parent_service` vs the prior
  derived-`parent_services` hypothesis.

## Phase deltas vs the original 006 plan (what this re-plan changes)

1. **T020 NAT toggle — UNGATED, now in scope as core work.** Official op:
   `PATCH /api/v1/routers/{router_id}/networks/{network_name}/nat` body
   `NatIn{nat_ip}` (enable) + `DELETE` (disable). Implement `convergeNAT` in
   `router_external.go` and the NAT rows of `isRouterUpToDate`. Live-validated:
   manually enabling NAT in the dashboard converged the router and stopped the
   reconcile loop — the convergence logic is proven; only the toggle call is missing.
   **This is the blocker for US3** (NAT egress) and for the perpetual NAT-drift loop.

2. **T021 resize — RESOLVED as no-op.** `RouterEdit` (PATCH body) is name+comment
   only; no upstream resize op exists. The immutable-reject (FR-002a fallback) is the
   **permanent** behavior. Mark T021 done; no code beyond what T017 already ships.

3. **Client regen reconciliation.** `make generate-client` against the official spec
   yields `NatIn`, `RouterEdit`, `RouterIn.parent_service`, and the
   `{network_name}` param. Reconcile: confirm NAT/detach key on name vs id;
   re-evaluate US3 binding via explicit `parent_service` at create. `availability_zone`
   is now official (drop that hand-patch). New extras (`/dnat-rules`,
   `/static-routes`, `/statistics`) stay **out of scope**.

4. **US3 — KEEP the explicit `publicIP` flag** (revised; Azure-AKS precedent +
   explicit-intent). `public_ip_enabled` re-added to the spec as a hand-patch; the
   MR `publicIP *bool` + CEL stay; `nodepool_external.go` keeps mapping it. Router-NAT
   + default route is the egress path. Live re-probe (bundle 19) confirms
   `publicIP:false` ⇒ no public address; fall back to placement only if ignored.

5. **Router behavior fixes (found live):**
   - **F-6** — Observe lacks the `starting` short-circuit that Update has → no-op
     reconcile loop while `starting`; mirror the guard in Observe.
   - **F-7** — router `no_paid` surfaces as `status:"error"`; map to PaymentRequired
     (not the generic UpstreamFailed "delete and recreate").
   - **F-8** — Router.Delete must **detach networks before deleting the router**,
     else it strands them as non-deletable `type:bgp` VPCs (409 "Network cannot be
     deleted"). One such orphan (`e2e-plan-probe-net2`) is permanently stuck →
     Timeweb support ticket.
   - **F-5** — FloatingIP router binding is UUID-keyed; the int64 `resourceID`/
     `BOUND-TO` can't render it. Add a string binding identifier (decode
     `AsFloatingIpResourceId1`) + add `router` to the `resourceType` doc enum.
   - **F-9** — FloatingIP create intermittently exceeds the 30s client timeout on
     slow ru-3 allocation; self-heals on retry (optional: 60s create timeout).

6. **e2e harness (T028 still OPEN):**
   - **(a) Bundle-18 hang** — the router-lifecycle bundle does **not complete** even
     when all three MRs are Ready=True/Synced=True and the router has fully settled
     (NAT loop already stopped). NOT the NAT loop. Investigate kuttl assert behavior
     vs a churning resourceVersion vs log buffering. **Blocks T028 bundle 18.**
   - **(b)** Router provisioning is 10–20 min/variable → bundle-18 `01-assert`
     timeout bumped 600→1800.
   - **(c)** Added events + provider-log collection to `kuttl.sh`
     (`test/e2e/.diagnostics/<ts>/`).
   - **(d)** Full-suite concurrent runs stack Timeweb's month-in-advance billing →
     `no_paid` on later resources; run router/private bundles scoped, not in the
     full concurrent suite.
